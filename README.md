# dotfiles
Personal macOS dotfiles managed with [GNU Stow](https://www.gnu.org/software/stow/).

## Fresh install
```bash
curl -fsSL https://raw.githubusercontent.com/yourusername/dotfiles/main/setup.sh | bash
```

## How it works
The repo is a flat mirror of your home directory. Stow symlinks everything into `$HOME`, so:

```
~/dotfiles/.zshrc           →  ~/.zshrc
~/dotfiles/.config/nvim/    →  ~/.config/nvim/
~/dotfiles/.config/tmux/    →  ~/.config/tmux/
```

Editing a config file in either location edits the same file - they're the same thing via symlink. Changes are tracked automatically in the repo.

## Updating dotfiles

### Editing an existing config
Just edit the file normally, wherever is convenient. Since everything is symlinked, changes are immediately reflected in `~/dotfiles`. Then commit:

```bash
cd ~/dotfiles
git add .
git commit -m "feat: update nvim config"
git push
```

### Adding a new config file
Nove the config into the repo at the matching path, then re-stow:

```bash
# Example: adding a new starship config:
mv ~/.config/starship.toml ~/dotfiles/.config/starship.toml

cd ~/dotfiles
stow .
git add .
git commit -m "feat: add starship config"
git push
```

### Re-stowing after pulling changes on another machine
If you've pulled changes from another machine that added new files:

```bash
cd ~/dotfiles
git pull
stow .
```

 Stow will create any new symlinks and leave existing ones untouched.

 ### Stow conflicts
 If a file already exists at the target location (i.e. not yet a symlink), Stow will refuse to overwrite it. Use `--adopt` to pull the existing file into the repo first:

 ```bash
 cd ~/dotfiles
 stow --adopt .
 git diff
 ```

 ## SSH keys
 Keys are generated by setup.sh but managed manually after that. Private keys are **never committed to this repo**, only `~/.ssh/config/` is tracked.

 ### Keys created by setup.sh
 | File | Purpose |
|---|---|
| `~/.ssh/id_github` | GitHub access |
| `~/.ssh/id_remotes` | General remote machines |

### Adding a new SSH key
```bash
# Generate the key
ssh-keygen -t ed25519 -C "description" -f ~/.ssh/id_newkey

# Add to the macOS keychain so it persists across reboots
ssh-add --apple-use-keychain ~/.ssh/id_newkey
```

Then add an entry to `~/.ssh/config/` (which lives at `~/dotfiles/.ssh/config`):

```
Host hostname-or-pattern
    IdentityFile ~/.ssh/id_newkey
    AddKeysToAgent yes
    UseKeychain yes
    User yourusername          # optional, omit if not needed
```

### Commit the config file change:
```bash
cd ~/dotfiles
git add .ssh/config
git commit -m "feat: add ssh key for <purpose>"
git push
```

The private key (`id_newkey`) stays on disc only, never add it to git.

### Copying a key to a remote machine
```bash
ssh-copy-id -i ~/.ssh/id_newkey.pub user@remote-host
```

## Adding or removing Homebrew packages
The package lists live in `setup.sh`. To keep things in sync after manually installing something:

```bash
# Install the package as normal
brew install ripgrep

# Then add it to the relevant list in setup.sh and commit
cd ~/dotfiles
git add setup.sh
git commit -m "chore: add ripgrep to formulae"
git push
```
